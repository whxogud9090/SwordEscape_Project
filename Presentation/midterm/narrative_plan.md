# Sword Escape Midterm Presentation Plan

## Audience
- 게임엔진입문 중간고사 발표를 듣는 교수님과 수강생

## Objective
- 2D 타일맵 기반 플랫폼 게임 `Sword Escape`의 핵심 컨셉, 스테이지 구성, 구현 기능, 개발 과정, 사용 리소스를 5분 안에 명확하게 설명한다.

## Narrative Arc
1. 게임의 정체성과 한 줄 소개를 먼저 보여준다.
2. 메인 메뉴와 조작 방법으로 플레이 진입 방식을 설명한다.
3. 스테이지 1~5의 흐름과 난이도 상승 구조를 요약한다.
4. 최소 요구사항 외에 추가한 기능으로 완성도를 강조한다.
5. 구현 근거, 사용 리소스, 개발 과정, 제출물 안내로 마무리한다.

## Slide List
1. 표지: 게임 제목과 핵심 컨셉
2. 메인 메뉴 화면과 포함 기능
3. 조작 방법과 핵심 시스템
4. 스테이지 1~2 구성 설명
5. 스테이지 3~4 구성 설명
6. 스테이지 5 보스전 설명
7. 추가 구현 요소
8. 개발 과정과 코드 설계 이유
9. 사용 리소스 및 제출물 정리
10. 마무리와 시연 포인트

## Source Plan
- 프로젝트 내부 에셋:
  - `Assets/Resources/MainMenu/moonlit-forest-gate-bg.png`
  - `Assets/sword.png`
  - `Assets/door.png`
  - `Assets/MR-Platformer-PixelAssets-v1/Main/Player/*.png`
  - `Assets/MR-Platformer-PixelAssets-v1/Main/Enemies/*.png`
- 프로젝트 코드:
  - 메인 메뉴, 스테이지 배경, 플레이어 이동, 아이템, 박쥐 추격, 이동 발판, 클리어/UI 관련 스크립트
- Git 정보:
  - 현재 이 환경에서는 `git` 명령 사용 불가. 발표 슬라이드에는 GitHub 링크/커밋 로그 캡처 삽입 위치를 마련하고 사용자가 최종 보완 가능하게 구성한다.

## Visual System
- 교수님 참고자료처럼 흰 배경, 상단 연회색 띠, 좌측 상단 청록/주황 포인트 바를 유지한다.
- 제목은 크고 명확하게, 본문은 2~4개의 핵심 문장만 넣는다.
- 게임 느낌을 살리기 위해 검, 문, 기사, 박쥐 스프라이트를 강조 오브젝트로 사용한다.
- 스테이지 소개 슬라이드는 카드형 요약과 작은 픽셀 오브젝트 배치로 구성한다.

## Asset Needs
- 게임 제목용 배경 1장
- 검, 문, 플레이어, 박쥐, 적 스프라이트
- 추가 캡처 없이도 성립하는 발표용 요약형 슬라이드

## Editability Plan
- 제목, 본문, 카드, 표시는 모두 편집 가능한 PowerPoint 텍스트/도형으로 만든다.
- 이미지는 장식과 분위기 전달용으로만 사용한다.
